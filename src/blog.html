The Markup Is the Music

A piece of music you can open in a text editor. One player, seven chapters: scroll, and the song on the right rebuilds itself in place.

The premise: a note is a <b> element placed on a CSS grid. The grid row is pitch, one row per semitone. The grid column is time, one column per sixteenth note. Tempo, instruments, mix, and arrangement are custom properties. The only JavaScript is a player that reads the rendered geometry — a note sounds exactly where the browser painted it — and an editor that writes inline styles back. There is no other data model.

That decision means the entire CSS engine becomes a music system, and it is also why this page works the way it does: press play once in the player, then just scroll. Each chapter swaps new markup and CSS into the same live document, and the transport never stops; the song simply becomes the next idea mid-loop.

hello, note

The smallest possible song. One track (a <section>), one note (a <b>), placed with grid-area: row 24 is middle C, columns 1 through 17 hold it for the whole bar. No song-specific CSS exists at this point. The markup alone is the music.

<main id="roll" style="--steps:16">
  <section class="trk" data-name="Note" style="--wave:sine;--vol:0.4">
    <b style="grid-area:24/1/25/17"></b>
  </section>
</main>

rows are pitch, columns are time

A melody is elements at different rows and columns. Up is higher; right is later. You could write a tune in any text editor, and the browser would render the score with no JavaScript at all. The script only adds sound and a mouse.

<b style="grid-area:24/1/25/5"></b>   <!-- C4, beat one -->
<b style="grid-area:20/5/21/9"></b>   <!-- E4, beat two -->
<b style="grid-area:17/9/18/13"></b>  <!-- G4, beat three -->
<b style="grid-area:12/13/13/17"></b> <!-- C5, beat four -->

tracks are sections, sound is properties

A bass arrives, and with it the instrument rack: custom properties on each track element. --wave chooses the voice (oscillators, wavetables like ep and brass, drum synthesis, even url(sample.wav)); --cutoff, --spread, --attack, --verb shape it; --bpm lives on <html>. The player reads computed values continuously, so devtools is a synthesizer.

<html style="--bpm:120">
...
<section class="trk" data-name="Melody" style="--wave:pulse25;--cutoff:2400">
<section class="trk" data-name="Bass"   style="--wave:triangle;--vol:0.3">

the cascade is a mixing console

Because volume is a property, muting is a selector. Because pitch is geometry, transposition is a transform: one row is a semitone, so one pixel is about six cents, and the format is natively microtonal. The checkboxes now sitting in the player are plain HTML inputs wired through :has() — try them. And since translate composes with transform, octave-up and quarter-tone-sharp stack.

body:has(#mute-bass:checked) .trk[data-name="Bass"] { --vol: 0 !important; }
body:has(#octave:checked)  .trk[data-name="Melody"] { transform: translateY(calc(var(--rh) * -12)); }
body:has(#quarter:checked) .trk[data-name="Melody"] { translate: 0 calc(var(--rh) * -0.5); }

media queries are arrangements

If the mix is CSS, the song can respond to anything CSS can see — including the width of its own window. Watch the player: while you read this chapter the page breathes the frame open and closed, and every time it narrows past the breakpoint the media query flips — drums cut, tempo drops — then the groove returns as it widens. The same file is a different arrangement on a phone.

@media (max-width: 480px) {
  html { --bpm: 84 !important; }
  .trk[data-name="Drums"] { --vol: 0 !important; }
}

animations are automation

Synth parameters are registered properties, so @keyframes are automation lanes. The music has changed underneath you: a bassline whose filter is swept by an animation — acid as a stylesheet rule — under a kick that pumps everything else through one property, --duck. Tick tape in the player: an animated transform warbles every track's pitch, and because the engine keeps re-reading geometry while notes sound, held notes bend live.

html.playing .trk[data-name="Acid"] { animation: sweep 3.1s ease-in-out infinite; }
@keyframes sweep { 0%, 100% { --cutoff: 250; } 50% { --cutoff: 2600; } }

html { --duck: 0.4; }  /* every kick ducks everything else */

music from nowhere

Two endgames at once. The first track's markup is eight empty elements; every pitch and position comes from a single rule built on sibling-index() and mod(). The stylesheet is the composer. The second track holds no notes at all until you tick summon in the player — it hears a track's ::before and ::after too, so a checkbox conjures music that exists nowhere in the document tree.

.trk[data-name="Formula"] b {
  grid-column: calc((sibling-index() - 1) * 2 + 1);
  grid-row: calc(28 - mod(sibling-index() * 5, 12));
}
body:has(#summon:checked) .trk[data-name="Ghost"]::before {
  content: ""; grid-area: 36 / 1 / 37 / 17;
}

where this goes

Stack all of it — per-note velocity, swing as fractional-step transforms, just-intonation detunes, samples baked in as data URIs, a data-pass attribute that turns loop counts into song structure — and you get records. Each is still a single HTML file you can open, edit, and git diff:

  • PUMP — filter house with a sampled 909 kit, a sidechain pump, and an eight-pass arrangement written in attribute selectors
  • HAMMERS X — 7,291 notes, every hit a microtonal cluster; acid, tape and cathedral as checkboxes
  • BLUE 31 — microtonal jazz: quarter-tone blue notes, septimal sevenths, and a quantize toggle that strips the humanity off with one rule
  • SELECTOR — acid techno whose markup contains zero notes; the composition is selector arithmetic
  • GHOST RATIO — the harmonic series in just intonation, with a scaleY animation that folds the whole spectrum into unison

Party trick: sonify.js in this repo is a console paste that reads any website's layout as one of these scores. Left edge is time, height is pitch, tags are instruments. Every page already is one of these songs. Nobody had pressed play.

If pitch is geometry and the mix is the cascade, then everything CSS can do — respond, animate, compose — music can do too. The stylesheet was a sequencer all along.
