The Markup Is the Music

A piece of music you can open in a text editor. Seven small songs, each one a complete instrument, each one explaining itself.

The premise: a note is a <b> element placed on a CSS grid. The grid row is pitch — one row, one semitone. The grid column is time — one column, one sixteenth note. Tempo, instruments, mix, and arrangement are custom properties. The only JavaScript is a player that reads the rendered geometry — a note sounds exactly where the browser painted it — and an editor that writes inline styles back. Saving a song re-serializes the document. There is no other data model. The file you are looking at is the file you are hearing.

That one decision — playback reads computed style and geometry, never a hidden JSON — means the entire CSS engine becomes a music system. What follows climbs that ladder, from a single note to music that exists only in selectors. Press play in each piece as you go; stop the last one first, or let them overlap, which is its own kind of piece.

hello, note

The smallest possible song. One track (a <section>), one note (a <b>), placed with grid-area: row 24 is middle C, columns 1 through 17 hold it for the whole bar. There is no song-specific CSS at all — the markup alone is the music.

<main id="roll" style="--steps:16">
  <section class="trk" data-name="Note" style="--wave:sine;--vol:0.4;--hue:200">
    <b style="grid-area:24/1/25/17"></b>
  </section>
</main>

Drag on the grid to draw more notes — the editor writes the same markup you just read. open the file ↗

rows are pitch, columns are time

A melody is elements at different rows and columns. Up is higher; right is later. That is the entire file format — you can write a tune in any text editor, and the browser renders the score with no JavaScript at all. The script only adds sound and a mouse.

<b style="grid-area:24/1/25/5"></b>   <!-- C4, beat one -->
<b style="grid-area:20/5/21/9"></b>   <!-- E4, beat two -->
<b style="grid-area:17/9/18/13"></b>  <!-- G4, beat three -->
<b style="grid-area:12/13/13/17"></b> <!-- C5, beat four -->

tracks are sections, sound is properties

Instruments are not configuration — they are custom properties on the track element. --wave chooses the voice (oscillators, wavetables like ep and brass, drum synthesis, even url(sample.wav)); --cutoff, --spread, --attack, --verb shape it; --bpm lives on <html>. Change any of them in devtools while it plays — the player reads computed values continuously.

<html style="--bpm:120">
...
<section class="trk" data-name="Melody" style="--wave:pulse25;--vol:0.2;--cutoff:2400">
<section class="trk" data-name="Bass"   style="--wave:triangle;--vol:0.3">

the cascade is a mixing console

Because volume is a property, muting is a selector. Because pitch is geometry, transposition is a transform — one row is a semitone, so one pixel is about six cents, which makes the format natively microtonal. And since translate composes with transform, the toggles below stack: an octave up and a quarter-tone sharp, if you like your thirds blue.

body:has(#mute-bass:checked) .trk[data-name="Bass"] { --vol: 0 !important; }
body:has(#octave:checked)  .trk[data-name="Melody"] { transform: translateY(calc(var(--rh) * -12)); }
body:has(#quarter:checked) .trk[data-name="Melody"] { translate: 0 calc(var(--rh) * -0.5); }

No JavaScript is involved in those checkboxes. It is only the cascade. open the file ↗

media queries are arrangements

If the mix is CSS, the song can respond to anything CSS can see — including the width of its own window. This embed sits in a resizable frame: drag its bottom-right corner narrower while it plays, and the drums dissolve while the tempo settles. The same file is a different arrangement on a phone.

@media (max-width: 560px) {
  html { --bpm: 84 !important; }
  .trk[data-name="Drums"] { --vol: 0 !important; }
}

animations are automation

Synth parameters are registered properties, so @keyframes are automation lanes. This bassline's filter is swept by an animation — acid as a stylesheet rule — and the kick pumps everything else through one property, --duck. The tape toggle animates a transform on every track: pitch warble — and because the engine keeps re-reading geometry while notes sound, held notes bend live.

html.playing .trk[data-name="Acid"] { animation: sweep 3.1s ease-in-out infinite; }
@keyframes sweep { 0%, 100% { --cutoff: 250; } 50% { --cutoff: 2600; } }

html { --duck: 0.4; }  /* every kick ducks everything else */

music from nowhere

Two endgames at once. The first track's markup is eight empty elements — every pitch and every position comes from a single rule built on sibling-index() and mod(): the stylesheet is the composer. And the second track holds no notes at all until you tick the box — the player also hears a track's ::before and ::after, so a checkbox can summon music that exists nowhere in the document tree.

.trk[data-name="Formula"] b {
  grid-column: calc((sibling-index() - 1) * 2 + 1);
  grid-row: calc(28 - mod(sibling-index() * 5, 12));
}
body:has(#summon:checked) .trk[data-name="Ghost"]::before {
  content: ""; grid-area: 36 / 1 / 37 / 17;
}

where the ladder goes

Stack all of it — per-note velocity, swing as fractional-step transforms, just-intonation detunes, samples baked in as data URIs, a data-pass attribute that turns loop counts into song structure — and you get records. Each one is still a single HTML file you can open, edit, and git diff:

  • PUMP — filter house with a sampled 909 kit, a sidechain pump, and an eight-pass arrangement written in attribute selectors
  • HAMMERS X — 7,291 notes, every hit a microtonal cluster; acid, tape and cathedral as checkboxes
  • BLUE 31 — microtonal jazz: quarter-tone blue notes, septimal sevenths, and a quantize toggle that strips the humanity off with one rule
  • SELECTOR — acid techno whose markup contains zero notes; the composition is selector arithmetic
  • GHOST RATIO — the harmonic series in just intonation, with a scaleY animation that folds the whole spectrum into unison

Party trick: sonify.js in this repo is a console paste that reads any website's layout as one of these scores — left edge is time, height is pitch, tags are instruments. Every page already is one of these songs. Nobody had pressed play.

If pitch is geometry and the mix is the cascade, then everything CSS can do — respond, animate, compose — music can do too. The stylesheet was a sequencer all along.
