The Markup Is the Music

midiCSS — a DAW where the HTML/CSS file is the song format. Every player below is a complete, self-contained instrument: the embed is the song is the editor. Press ▶ in each one (and stop the last one, or don't).

The premise: a note is a <b> element placed on a CSS grid. The grid row is pitch (one row = one semitone), the grid column is time (one column = a 16th note). Tempo, instruments, mix, and arrangement are CSS custom properties. The only JavaScript is a player that reads the rendered geometry — a note sounds exactly where the browser painted it — and an editor that writes inline styles back. Saving a song re-serializes the document. There is no other data model. The file you're looking at is the file you're hearing.

That one decision — playback reads computed style and geometry, not some hidden JSON — means the entire CSS engine becomes a music system. This post climbs that ladder, from one note to music that exists only in selectors.

01hello, note

The smallest possible song. One track (a <section>), one note (a <b>), placed with grid-area: row 24 is C4, columns 1 through 17 make it a whole bar long. No song-specific CSS at all.

<main id="roll" style="--steps:16">
  <section class="trk" data-name="Note" style="--wave:sine;--vol:0.4;--hue:200">
    <b style="grid-area:24/1/25/17"></b>
  </section>
</main>

02rows are pitch, columns are time

A melody is just elements at different rows and columns. Up is higher, right is later. This is the entire file format — you can write a tune in any text editor, and the browser renders the piano roll with zero JavaScript. The script only adds sound and mouse handling.

<b style="grid-area:24/1/25/5"></b>   <!-- C4, beat 1 -->
<b style="grid-area:20/5/21/9"></b>   <!-- E4, beat 2 -->
<b style="grid-area:17/9/18/13"></b>  <!-- G4, beat 3 -->
<b style="grid-area:12/13/13/17"></b> <!-- C5, beat 4 -->

03tracks are sections, sound is properties

Instruments aren't config — they're custom properties on the track element. --wave picks the voice (oscillators, wavetables like ep and brass, drum synthesis, even url(sample.wav)), and --cutoff, --spread, --attack, --verb, --echo shape it. --bpm lives on <html>. Change any of them in devtools while it plays; the player reads computed values continuously.

<html style="--bpm:120">
...
<section class="trk" data-name="Melody" style="--wave:pulse25;--vol:0.2;--cutoff:2400">
<section class="trk" data-name="Bass"   style="--wave:triangle;--vol:0.3">

04the cascade is a mixing console

Because volume is a property, muting is a selector. Because pitch is geometry, transposition is a transform — one row is a semitone, so one pixel is about six cents, which makes the format natively microtonal. And since translate composes with transform, the toggles below stack: octave up and a quarter-tone sharp.

body:has(#mute-bass:checked) .trk[data-name="Bass"] { --vol: 0 !important; }
body:has(#octave:checked)  .trk[data-name="Melody"] { transform: translateY(calc(var(--rh) * -12)); }
body:has(#quarter:checked) .trk[data-name="Melody"] { translate: 0 calc(var(--rh) * -0.5); }

No JavaScript involved in any of those checkboxes — it's the cascade. open this file directly ↗

05media queries are arrangements

If the mix is CSS, then the song can respond to anything CSS can see — including the viewport. This embed sits in a resizable wrapper: drag its right edge narrower while it plays and the drums fade out and the tempo drops. The same file is a different arrangement on a phone.

@media (max-width: 560px) {
  html { --bpm: 84 !important; }
  .trk[data-name="Drums"] { --vol: 0 !important; }
}

06animations are automation

Synth parameters are registered properties, so @keyframes are automation lanes. This bassline's filter is swept by an animation — acid as a stylesheet rule — and the kick pumps the whole mix through one property, --duck. The tape toggle animates a transform on every track: pitch warble, and because the engine re-reads geometry while notes sound, held notes bend live.

html.playing .trk[data-name="Acid"] { animation: sweep 3.1s ease-in-out infinite; }
@keyframes sweep { 0%, 100% { --cutoff: 250; } 50% { --cutoff: 2600; } }

html { --duck: 0.4; }  /* every kick ducks everything else */

07music from nowhere

Two endgames at once. The first track's markup is eight empty elements — every pitch and position below comes from one rule using sibling-index() and mod(): the stylesheet is the composer (Chrome 138+). And the second track has no notes at all until you tick the box: the player also reads a track's ::before and ::after as notes, so a checkbox can summon music that exists nowhere in the document tree.

.trk[data-name="Formula"] b {
  grid-column: calc((sibling-index() - 1) * 2 + 1);
  grid-row: calc(28 - mod(sibling-index() * 5, 12));
}
body:has(#summon:checked) .trk[data-name="Ghost"]::before {
  content: ""; grid-area: 36 / 1 / 37 / 17;
}

where this ladder goes

Stack all of it — plus per-note velocity, swing as fractional-step transforms, just-intonation detunes, samples as data URIs, a data-pass attribute that turns loop counts into song structure — and you get actual records, each one still a single HTML file you can open, edit, and git diff:

  • PUMP — filter house with a sampled 909 kit, sidechain pump, and an 8-pass arrangement written in attribute selectors
  • HAMMERS X — 7,291 notes; every hit a microtonal cluster; acid, tape and cathedral as checkboxes
  • BLUE 31 — microtonal jazz: quarter-tone blue notes, septimal sevenths, and a quantize toggle that strips the humanity off with one rule
  • SELECTOR — acid techno whose markup contains zero notes: the entire composition is selector math
  • GHOST RATIO — the harmonic series in just intonation, with a scaleY animation that folds the spectrum into unison

Party trick: sonify.js in this repo is a console paste that treats any website's layout as one of these scores — left edge is time, height is pitch, tags are instruments. Every page already is a midiCSS song; nobody's pressed play on them.

Everything here is one repo: strict-TS player (~700 lines), a build script that composes songs as data, and the songs themselves. The format is the thesis: if pitch is geometry and mix is the cascade, then everything CSS can do — respond, animate, compose — music can do too.
